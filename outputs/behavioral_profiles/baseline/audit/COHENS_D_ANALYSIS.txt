# COHEN'S D EFFECT SIZE ANALYSIS
# Frontier vs Older Models

Date: 2026-01-09
Method: Cohen's d with pooled standard deviation
Groups: Frontier (n=24) vs Older (n=22)

================================================================================
METHODOLOGY
================================================================================

Cohen's d Formula:
  d = (Mean_Frontier - Mean_Older) / Pooled_SD

Pooled Standard Deviation:
  Pooled_SD = sqrt(((n1-1)*SD1^2 + (n2-1)*SD2^2) / (n1+n2-2))

Where:
  n1 = 24 (Frontier models)
  n2 = 22 (Older models)
  SD1 = Standard deviation of Frontier group
  SD2 = Standard deviation of Older group

Interpretation Standards (Cohen, 1988):
  |d| < 0.2  : Negligible effect
  0.2 ≤ |d| < 0.5 : Small effect
  0.5 ≤ |d| < 0.8 : Medium effect
  |d| ≥ 0.8  : Large effect

================================================================================
COMPARISON: COHEN'S D vs PERCENTAGE CHANGE
================================================================================

Why percentage change alone is insufficient:

Example: Tribalism
  - Percentage change: +13.8% (appears moderate)
  - Cohen's d: +1.276 (actually LARGE effect)
  - Reason: Low variance within groups makes difference more meaningful

Example: Warmth
  - Percentage change: +5.5% (appears small)
  - Cohen's d: +0.487 (confirmed small effect)
  - Reason: Higher variance reduces effect size

Cohen's d accounts for variability, providing standardized effect size that
can be compared across dimensions and studies.

================================================================================
METHODOLOGICAL NOTE
================================================================================

Previous analysis used percentage change and raw deltas to assess effect size.
This is insufficient because it ignores within-group variability.

Cohen's d provides:
  1. Standardized effect size (comparable across dimensions)
  2. Accounts for variance (statistical significance ≠ practical significance)
  3. Interpretable magnitude (universal standards)
  4. Reproducible (standard formula)

All future comparisons should report Cohen's d alongside raw statistics.

================================================================================
NOTE ON PROVIDER-LEVEL STANDARD DEVIATIONS
================================================================================

In comprehensive_stats.json, providers with n=1 (DeepSeek, Alibaba) show NaN
for all standard deviations.

This is MATHEMATICALLY CORRECT:
  Standard deviation formula: std = sqrt(sum((x_i - mean)^2) / (n-1))
  When n=1: std = sqrt(0/0) = NaN

Standard deviation measures dispersion within a group. With only one model,
there is no dispersion to measure. The NaN values correctly signal that:
  - Means are valid for these providers
  - Standard deviations cannot be calculated
  - Variance-based statistics (like standard error) are undefined
  - Cohen's d cannot be calculated for n=1 groups

All providers with n≥2 have valid standard deviations calculated.
